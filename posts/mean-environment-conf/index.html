最近打算尝试一下新的技术栈，先做一个个人网站练手。后端方面选择的是NodeJS，框架用Express，前端方面选择的是Angular, 数据库选择的是常与node搭配的MongoDB，并决定开始使用TypeScript。由于Github上找到的手脚架都不太符合我的需要，于是决定自己搭建，记录一下，避免日后踩坑。

顺便提一句，为了契合TypeScript，我把编辑器从Sublime Text换成了VSCode, 这几天用下来，体验不错。自身集成的Git和Terminal用起来很方便，安装插件也比Sublime更简单快捷。

初始项目结构

Node环境啥的不用说了，先再项目根目录下运行

npm init

生成 package.json 文件，然后新建两个文件夹，一个叫 client 用来放Angular的文件，一个叫 server 用来放Express的文件。

安装Angular

这一步很简单，按照官网的做就可以了。先安装Angluar的CLI.

npm install -g @angular/cli

然后新建项目并运行

ng new client

cd client
ng serve

打开 http://localhost:4200 可以查看运行结果。

安装Express

准备

首先初始化目录，其中 src 用来存放项目的主要代码

cd server
npm init

mkdir src

之后安装Express相关依赖，其中 dotenv 用来加载 .env 文件中的环境变量

npm install express body-parser dotenv --save

配置TypeScript

Angular的项目代码本身就是TypeScript写的，所以客户端不需要配置，服务器端需要配置一下。

首先安装相关依赖

npm install -D tsnode tslint typescript --save-dev

新建文件 tsconfig.json , 进行typescript相关配置

{
  "compilerOptions": {
    "target": "es6",
    "module": "commonjs",
    "outDir": "dist",
    "esModuleInterop": true,
    "noImplicitAny": true,
    "sourceMap": true
  },
  "include": [
    "src/**/*.ts"
  ],
  "exclude": [
    "node_modules"
  ]
}

为了让ts可以使用我们安装的依赖，还需要安装相应的类型定义依赖 npm install @types/[module_name] --save-dev

之后再运行 tsc 即可将 src 目录下的ts文件输出为js文件到dist文件夹中

连接客户端

新建一个 .env 文件，输入环境变量

# 运行端口
PORT=3000

进入 src , 在该目录下新建文件 index.ts

// server/src/index.ts
import express from 'express';
import bodyParser from 'body-parser';
import path from 'path';

const app = express();

// Parsers
app.use(bodyParser.json());
app.use(bodyParser.urlencoded({
  extended: false
}));

// Angular DIST output folder
app.use(express.static(path.join(__dirname, '../../client/dist')));

// Send all other requests to the Angular app
app.get('*', (req, res) => {
  res.sendFile(path.join(__dirname, '../../client/dist/index.html'));
});

export default app;

回到 server 目录，新建文件 index.ts

// server/index.ts
import app from './src';
import dotenv from 'dotenv';

// Load environment variables from .env file
dotenv.config({ path: '../.env' });

const port = process.env.PORT || '3000';

app.listen(port, () => console.log(`Running on localhost:${port}`));

配置package.json的scripts

之后配置一些命令，方便之后编译和测试。

/package.json

"scripts": {
    "app-install": "npm install && npm run install:client && npm run install:server",
    "install:client": "cd client && npm install",
    "install:server": "cd server && npm install",
    "start": "concurrently \"npm run start:client\" \"npm run start:server\"",
    "start:client": "cd client && ng serve",
    "start:server": "cd server && npm start",
    "start:prod": "npm run build && export NODE_ENV=production && cd server/dist && node index.js",
    "build": "concurrently \"npm run build:client\" \"npm run build:server\"",
    "build:client": "cd client && ng build --prod --aot",
    "build:server": "cd server && npm run build",
    "test": "echo \"Error: no test specified\" && exit 1"
}

server/package.json

"scripts": {
    "lint": "tslint --project tsconfig.json",
    "start": "npm run build:live",
    "build": "tsc",
    "build:live": "pm2 start ts-node -- index.ts",
    "test": "echo \"Error: no test specified\" && exit 1"
},

测试

在根目录下运行 npm run start , 打开 http://localhost:3000/ 查看结果应和端口4200的结果相同。

安装MongoDB

在 server 目录下，安装相关依赖，以下依赖同样需要相应的 @types/[module_name]

npm install mongodb mongoose bluebird --save

编写测试代码，在 server/index.ts 中添加如下代码

import mongoose from 'mongoose';
import bluebird from 'bluebird';

mongoose.Promise = bluebird;
mongoose.connect(process.env.MONGODB_URI_LOCAL, { useMongoClient: true, promiseLibrary: require('bluebird') })
  .then(() =>  console.log('connection succesful'))
  .catch((err) => console.error(err));

如果运行时，控制台打印 connection succesful , 说明数据库连接成功。
